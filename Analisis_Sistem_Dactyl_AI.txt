--------------------------------------------------------------------------------
1. RINGKASAN EKSEKUTIF (EXECUTIVE SUMMARY)
--------------------------------------------------------------------------------
Dactyl-AI adalah inovasi sistem Human-Computer Interaction (HCI) tanpa sentuhan 
(touchless interface) yang memungkinkan pengguna mengontrol kursor mouse, mengetik 
pada keyboard virtual, mengatur multimedia, hingga melakukan navigasi sistem 
operasi hanya menggunakan gestur tangan di depan kamera web (webcam).

Sistem ini memecahkan masalah ketergantungan pada perangkat keras fisik (mouse & 
keyboard) serta masalah oklusi (jari saling menutupi) dengan menerapkan konsep 
"Dual-Wielding / Bimanual Interface". Tangan penunjuk navigasi dan tangan pengontrol 
fungsi bekerja secara simultan maupun eksklusif yang diproses melalui kombinasi 
Computer Vision (MediaPipe Hands), Machine Learning (Support Vector Machine), 
serta Rule-Based Geometrical Logic.


--------------------------------------------------------------------------------
2. ARSITEKTUR & TEKNOLOGI PENDUKUNG
--------------------------------------------------------------------------------
Sistem dibangun di atas ekosistem Python 3.8+ dengan arsitektur multimodul:
a. MediaPipe Hands (Google) : Berfungsi sebagai pemindai (feature extractor) 
   yang mendeteksi hingga 2 tangan secara simultan dan mengekstrak 21 titik sendi 
   (hand-landmarks) koordinat spasial 3D (X, Y, Z) per tangan.
b. Scikit-Learn (SVM & Scaler) : Mesin kecerdasan buatan (AI Brain) yang 
   mengklasifikasikan 126 atribut koordinat spasial ke dalam 8 kelas gestur.
c. OpenCV (Computer Vision) : Menangani akuisisi video secara real-time, 
   mirroring frame, rendering antarmuka visual (hologram UI, progress bar, 
   bingkai kamera), serta manipulasi gambar.
d. PyAutoGUI : Mengonversi instruksi gestur menjadi event sistem operasi 
   seperti gerakan mouse, klik, drag, scroll, keystroke keyboard, dan shortcut.
e. Win32 API (ctypes & wintypes) : Mengatur window aplikasi agar bersanad 
   Always on Top (Topmost), semi-transparan (60% opacity), dan memosisikan diri 
   di pojok kanan bawah monitor agar tidak mengganggu area kerja pengguna.
f. PowerShell Asynchronous : Digunakan untuk pengontrolan kecerahan monitor 
   (brightness) di latar belakang tanpa menyebabkan lag/bottleneck pada video feed.


--------------------------------------------------------------------------------
3. ALUR KERJA SISTEM (REAL-TIME PIPELINE)
--------------------------------------------------------------------------------
Secara garis besar, pipeline pemrosesan data berjalan dalam 5 tahap setiap frame:

[1. Akuisisi & Preprocessing Video]
    - Webcam menangkap frame video (resolusi 1280x720).
    - Frame dicerminkan (cv2.flip) agar interaksi terasa alami seperti berkaca.
    - Konversi ruang warna dari BGR (OpenCV default) ke RGB (standar MediaPipe).

[2. Ekstraksi Fitur Spasial (MediaPipe)]
    - Mendeteksi keberadaan tangan kanan dan tangan kiri.
    - Ekstraksi 21 koordinat (X, Y, Z) per tangan. Jika satu tangan tidak 
      terdeteksi, koordinatnya diisi nilai nol (0.0).
    - Total data yang dihasilkan adalah vektor 126 dimensi (63 Kiri + 63 Kanan).
    - Karena efek cermin (flip), sistem melakukan koreksi label otomatis 
      (MediaPipe 'Left' -> Tangan Kanan fisik, MediaPipe 'Right' -> Tangan Kiri fisik).

[3. Normalisasi & Klasifikasi Gestur (AI Brain)]
    - Vektor 126 dimensi dinormalisasi menggunakan StandardScaler agar rentang 
      nilainya seragam.
    - Model Support Vector Machine (SVM) dengan Kernel RBF memprediksi label 
      gestur (Kelas 0 hingga Kelas 7).

[4. Eksekusi Logika Kontrol & Aksi Sistem]
    - Berdasarkan label gestur dari SVM, sistem mengevaluasi aturan geometri 
      (Euclidean distance antar jari) dan kalkulasi temporal (lama penahanan gestur).
    - Perintah dikirimkan ke OS melalui PyAutoGUI, Win32 API, atau PowerShell.

[5. Rendering Visual & UI Feedback]
    - Menampilkan status gestur, titik hover keyboard, progress bar interaktif, 
      dan indikator mode di atas layar kamera yang berjalan pada 24-30 FPS.


--------------------------------------------------------------------------------
4. DAFTAR KELAS GESTUR & MEKANISME KERJA (LABEL 0 - 7)
--------------------------------------------------------------------------------
Sistem membagi instruksi ke dalam 8 label gestur yang memiliki logika khusus:

• LABEL 0 : TELUNJUK LURUS (Move / Navigation Mode)
  - Fungsi  : Menggerakkan kursor mouse di layar monitor.
  - Kerja   : Menggunakan koordinat ujung jari telunjuk (Landmark 8) tangan kanan.
  - Optimasi: Dilengkapi algoritma "Heuristic Jitter Filtering" menggunakan 
    Exponential Moving Average (EMA) dengan alpha = 0.25. Filter ini meredam 
    getaran alami tangan (tremor) sehingga pergerakan kursor sangat mulus (smooth).

• LABEL 1 : CUBIT / PINCH (Click & Drag Mode)
  - Fungsi  : Melakukan Klik Kiri, Klik Kanan, dan Tahan-Seret (Hold-Drag).
  - Kerja   : Dihitung menggunakan jarak Euclidean antara ujung jempol (Landmark 4) 
    dan ujung jari lainnya pada tangan kanan (Threshold jarak < 0.045).
    * Klik Kiri   : Cubit jempol + telunjuk dilepas dengan cepat (< 0.3 detik).
    * Hold-Drag   : Cubit jempol + telunjuk ditahan (> 0.3 detik). Sistem memicu 
                    event `mouseDown` dan memungkinkan pengguna menyeret jendela/item.
    * Klik Kanan  : Cubit jempol + jari tengah kanan (dengan cooldown 0.4 detik).

• LABEL 2 : MEGANG BOLA / C-SHAPE (Multimedia & Brightness Control)
  - Fungsi  : Mengatur volume suara dan kecerahan layar monitor secara intuitif.
  - Kerja   : Menggunakan kalkulasi sudut rotasi pergelangan tangan (Landmark 0) 
    terhadap jari tengah (Landmark 9) dengan fungsi matematika `atan2`.
    * Volume Control     : Eksklusif Tangan Kanan. Memutar tangan searah/berlawanan 
                           jarum jam mengubah volume sistem (volumeup / volumedown).
    * Brightness Control : Eksklusif Tangan Kiri. Memutar tangan kiri memicu 
                           thread PowerShell asynchronous (WmiSetBrightness) untuk 
                           mencerahkan atau meredupkan monitor tanpa membuat video lag.

• LABEL 3 : PEACE / DUAL V-SIGN (Hologram Keyboard Summoning)
  - Fungsi  : Memunculkan dan menyembunyikan Keyboard Virtual Transparan.
  - Kerja   : Pengguna menahan gestur Peace (V-Sign) dengan kedua tangan selama 
    2 detik. Keyboard hantu (3 baris QWERTY) akan muncul di bagian bawah layar.
    * Mengetik dilakukan dengan mengarahkan (hover) jari telunjuk ke kotak tombol 
      selama 1.0 detik. Bar progres hijau akan terisi penuh sebelum karakter diketik.
    * Tombol khusus tersedia untuk Space, Backspace, Enter, dan Close.
    * Saat keyboard aktif, aksi navigasi mouse dibekukan agar tidak terjadi salah klik.

• LABEL 4 : FRAME KAMERA (Screenshot Area Mode)
  - Fungsi  : Mengambil tangkapan layar (screenshot) pada area yang disesuaikan.
  - Kerja   : Pengguna membentuk bingkai kotak menggunakan jempol dan telunjuk dari 
    kedua tangan. Sistem menghitung Bounding Box (x_min, y_min, x_max, y_max).
    * Jika bingkai ditahan selama 1.5 detik (terdapat visual countdown bar & bingkai 
      sudut hijau), sistem memetakan koordinat kamera ke resolusi monitor.
    * Screenshot dipotong sesuai area bingkai, disimpan ke folder `Data/screenshots/`, 
      dan disertai efek visual kamera flash putih pada layar.

• LABEL 5 : DUA JARI LURUS NINJA (Scroll Mode)
  - Fungsi  : Melakukan gulir layar vertikal (naik/turun) dan horizontal (kiri/kanan).
  - Kerja   : Posisi jari tengah (Landmark 9) dilacak pergerakan delta-nya (dx, dy).
    Pergerakan vertikal memicu `pyautogui.scroll`, sedangkan horizontal memicu 
    `pyautogui.hscroll` dengan faktor akselerasi yang responsif.

• LABEL 6 : TANGAN TERBUKA KERTAS (Swipe Tab / Task Switcher)
  - Fungsi  : Berpindah antar jendela aplikasi (Alt + Tab / Alt + Shift + Tab).
  - Kerja   : Eksklusif Tangan Kiri. Sistem mencatat history posisi X selama 
    beberapa frame untuk menghitung kecepatan geser (velocity = dx / dt).
    * Swipe Cepat ke Kanan (Velocity > 1.3)  : Memicu shortcut `Alt + Tab` (Next Tab).
    * Swipe Cepat ke Kiri (Velocity < -1.3) : Memicu `Alt + Shift + Tab` (Prev Tab).
    * Dilengkapi cooldown 1.2 detik agar tidak terjadi perpindahan tab tak terkendali.

• LABEL 7 : TANGAN MENGEPAL KUAT / FIST (Standby / Pause Lock Mode)
  - Fungsi  : Mengunci sistem (System Locked / Standby) saat pengguna beristirahat.
  - Kerja   : Genggam tangan (fist) selama 1.0 detik untuk mengunci sistem. Seluruh 
    kontrol mouse dan keyboard dimatikan, digantikan oleh overlay merah semi-transparan.
    * Untuk membuka kunci (Unlock), pengguna cukup menggenggam tangan kembali selama 
      1.0 detik. Fitur ini mencegah kursor bergerak liar saat pengguna minum atau duduk biasa.

5. FITUR UNGGULAN & KEUNGGULAN TEKNIS
--------------------------------------------------------------------------------
1. Dual-Wielding & Bimanual Task Division:
   Pembagian tugas yang jelas antara tangan kanan (navigasi & klik) dan tangan 
   kiri (shortcut, brightness, swipe tab, keyboard hovering) mencegah konflik 
   dan masalah oklusi visual pada kamera.

2. Heuristic Jitter Filtering (EMA Alpha=0.25):
   Pergerakan kursor mouse tidak mengambil data mentah dari MediaPipe yang cenderung 
   bergetar/tremor, melainkan disaring menggunakan Exponential Moving Average sehingga 
   kursor bergerak sangat mulus seperti menggunakan mouse fisik berpresisi tinggi.

3. Lightweight AI Engine (Vektor vs Image):
   Karena klasifikasi ML dilakukan pada data vektor koordinat numerik 1D (126 angka), 
   bukan pada pemrosesan gambar/citra 2D (seperti CNN), beban komputasi CPU/GPU 
   sangat ringan, memungkinkan program berjalan lancar di PC spesifikasi menengah.

4. Non-Intrusive Always-on-Top GUI (Win32 API):
   Window kamera aplikasi dimodifikasi pada tingkat sistem operasi Windows agar 
   selalu berada di lapisan teratas (Topmost) dengan transparansi 60%, serta berada 
   di pojok kanan bawah monitor sehingga tidak menutupi pekerjaan pengguna.

5. Asynchronous Background PowerShell Execution:
   Pengaturan kecerahan layar dilakukan secara asinkron menggunakan thread terpisah 
   agar pemanggilan WMI Windows tidak menyebabkan jeda frame (lag/stuttering) pada 
   tangkapan video webcam.


--------------------------------------------------------------------------------
6. STRUKTUR MODUL DAN STRATEGI PELATIHAN AI
--------------------------------------------------------------------------------
Project Dactyl-AI terdiri dari 3 modul utama dalam folder `src/`:

a. `data_collection.py`:
   - Digunakan untuk mengumpulkan dataset bimanual.
   - Mengambil 126 koordinat dari kamera dan menyimpannya ke `Data/dataset_bimanual.csv`.
   - Perekaman dikontrol dengan menahan tombol keyboard [0] sampai [7] saat berpose.

b. `train_model.py`:
   - Membaca dataset CSV dan membagi data menjadi 3 bagian: 80% Train, 10% Validation 
     (Fase Tuning), dan 10% Test (Fase Evaluasi Akhir) dengan teknik stratified sampling.
   - Melakukan normalisasi atribut menggunakan `StandardScaler`.
   - Melatih model Support Vector Machine (SVM) dengan parameter `kernel='rbf'`, 
     `C=10`, dan `gamma='scale'`.
   - Menyimpan otak AI (`model`) dan formula normalisasi (`scaler`) ke dalam format 
     pickle di file `models/svm_gesture_model.pkl`.

c. `run_system.py`:
   - Modul eksekusi utama yang memuat model AI dan scaler.
   - Menjalankan loop kamera real-time, mendeteksi landmark, memprediksi gestur, 
     dan mengeksekusi logika kontrol kursor, keyboard, serta multimedia.


--------------------------------------------------------------------------------
7. KESIMPULAN
--------------------------------------------------------------------------------
Dactyl-AI berhasil menggabungkan presisi ekstraksi koordinat MediaPipe, kehandalan 
klasifikasi Support Vector Machine (SVM), serta kecerdasan logika geometri spasial 
untuk menciptakan sistem kontrol antarmuka masa depan yang sepenuhnya tanpa sentuhan. 
Dengan performa real-time (24-30 FPS), fitur peredam getaran (jitter filter), dan 
desain antarmuka transparan yang ergonomis, sistem ini menjadi solusi Human-Computer 
Interaction yang sangat praktis, stabil, dan futuristik.
