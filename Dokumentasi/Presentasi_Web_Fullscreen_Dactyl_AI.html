Tantangan Konvensional
Keterbatasan Perangkat Mekanis
Mouse dan keyboard fisik membutuhkan kontak permukaan statis, rentan terhadap RSI (Repetitive Strain Injury), dan tidak higienis dalam lingkungan klinis/industri steril.
Kelemahan Sistem 2D/CNN
Beban Komputasi & Oklusi

Solusi Dactyl-AI V2.5
Bimanual Vector-Based AI
Menggabungkan pelacakan 3D spasial (MediaPipe) dengan klasifikasi vektor ringan (SVM 126D) yang sangat cepat di CPU biasa dan membagi peran kedua tangan secara asinkron.

Real-Time Processing Pipeline (5 Tahap)

Aliran pemrosesan dari tangkapan foton kamera hingga eksekusi perintah sistem operasi secara berurutan dalam latensi < 16.5 ms.

📷
1. Akuisisi & Preprocessing
Webcam 720p menangkap frame RGB. Dilakukan Mirror Flip (flip=1) agar interaksi terasa alami seperti cermin.
➡️
🖐️
2. Spasial Ekstraksi (3D)
MediaPipe mengekstrak koordinat $(X, Y, Z)$ dari 21 titik per tangan. Koreksi label cermin (Left $\rightarrow$ Right physical).
➡️
📐
3. Vektorisasi & Scaler
Penggabungan koordinat tangan Kiri ($63$) + Kanan ($63$) = vektor $126$ dimensi. Dinormalisasi via `StandardScaler`.
➡️
⚡
4. Prediksi SVM Brain
Model SVM RBF mengevaluasi vektor dan mengembalikan ID gestur ($0 - 7$) dengan threshold keyakinan $\ge 75\%$.
➡️
🖥️
5. Action Dispatcher
Eksekusi aksi sistem: EMA Cursor Move, Click, Scroll, Screenshot, atau Hologram Keyboard.

Konstruksi Matriks Fitur 126-Dimensi

Sistem menyusun vektor tunggal $X_{input}$ dengan menggabungkan koordinat tangan Kiri ($L$) dan Kanan ($R$): 

X_input = [
  L_x0, L_y0, L_z0, ..., L_x20, L_y20, L_z20,  # 63 fitur Kiri
  R_x0, R_y0, R_z0, ..., R_x20, R_y20, R_z20   # 63 fitur Kanan
] # Total: 126 elemen floating point

Zero-Padding Strategy: Jika pengguna hanya mengangkat satu tangan (misal hanya tangan kanan aktif), maka 63 fitur tangan kiri secara otomatis diisi nilai `0.0` agar dimensi input model tetap stabil $126$ fitur.

Otak AI: Support Vector Machine (SVM RBF)

Mengapa vektor 126-dimensi + SVM jauh lebih superior dibanding CNN gambar 2D untuk sistem kontrol kursor real-time?

Konfigurasi Hiperparameter Model (`SVC`)
  • `kernel='rbf'` (Radial Basis Function): Mampu memisahkan batas keputusan gestur tangan yang bersifat non-linear kompleks dalam ruang multi-dimensi dengan pemetaan kernel hiper-bola.
  • `C=10`: Parameter penalti regularization yang tinggi untuk meminimalkan misklasifikasi pada data training dan menjaga ketajaman pemisahan kelas gestur.
  • `class_weight='balanced'`: ENHANCED FIX Bobot otomatis penyeimbang kelas untuk mengatasi ketimpangan jumlah sampel pada dataset ($4.41 : 1$).
  • `probability=True`: Memungkinkan kalkulasi keyakinan (`predict_proba`) untuk menolak prediksi di bawah threshold $75\%$.

Spesifikasi Klasifikasi Gestur (Label 0 – 3)

Pemetaan logika aksi sistem operasi berbasis aturan untuk empat kelas gestur utama pertama.

Label 0 • 9.4% Data
Telunjuk Lurus (Move)
Aksi OS: `pyautogui.moveTo(x, y)`
Geometri: Ujung telunjuk kanan (Landmark 8) lurus, jari lain menekuk.
Filter: Exponential Moving Average (`alpha=0.25`) untuk menghapus jitter/getaran tangan.
Label 1 • 31.9% Data
Cubit / Pinch (Click & Drag)
Aksi OS: Click & Hold-Drag (`mouseDown`)
Geometri: Jarak 3D Jempol-Telunjuk < `0.045`.
Logika Temporal: Dilepas < 0.3s = Klik Kiri; Ditahan > 0.3s = Drag Kursor; Cubit Jari Tengah = Klik Kanan.
Label 2 • 11.2% Data
C-Shape (Volume & Brightness)
Aksi OS: Volume Up/Down & WMI Brightness
Geometri: Rotasi pergelangan `atan2(delta_y, delta_x)` dengan threshold delta > `0.08` rad.
Bimanual: Tangan Kanan = Volume, Tangan Kiri = Brightness (Asinkron).
Label 3 • 8.7% Data
Dual V-Sign (Hologram KB)
Aksi OS: Toggle Virtual Keyboard
Geometri: Kedua tangan mengacungkan V-Sign (Peace) secara simultan.
Logika Temporal: Ditahan selama 2.0 detik dengan anti-freeze non-blocking check.

Spesifikasi Klasifikasi Gestur (Label 4 – 7)

Pemetaan logika navigasi tingkat lanjut, screenshot bimanual, dan mekanisme pengamanan sistem.

Label 4 • 9.4% Data
Frame Kamera (Screenshot)
Aksi OS: `pyautogui.screenshot(region)`
Geometri: Bounding box dari ujung telunjuk & jempol kedua tangan membentuk bingkai.
Logika Temporal: Hitung mundur 1.5 detik dengan visual progress bar + animasi shutter flash.
Label 5 • 8.0% Data
Ninja Dua Jari (Scroll)
Aksi OS: `scroll` & `hscroll`
Geometri: Telunjuk & tengah sejajar.
Logika: Pergeseran vertikal `dy` atau horizontal `dx` > `0.015` dikalikan faktor akselerasi `2500` untuk scroll mulus ganda arah.
Label 6 • 7.2% Data
Swipe Kertas (Alt+Tab)
Aksi OS: `Alt+Tab` / `Alt+Shift+Tab`
Geometri: Tangan kiri terbuka (Palm) bergerak cepat.
Logika: Velocity `dx/dt > 1.3` pada history 6-frame dengan cooldown pengamanan 1.2 detik.
Label 7 • 14.3% Data
Mengepal / Fist (Standby)
Aksi OS: System Lock / Unlock
Geometri: Seluruh jari menggenggam erat (Fist ✊).
Logika Temporal: Ditahan 1.0 detik untuk membekukan semua input mouse dengan overlay merah peringatan.

Temuan Kritis: Imbalance Ratio 4.41 : 1

Kelas terbesar (Label 1 - Pinch, $525$ sampel) memiliki jumlah $4,41 \times$ lipat lebih banyak dibanding kelas terkecil (Label 6 - Swipe, $119$ sampel).

✅ SOLUSI REMEDIASI YANG TELAH DITERAPKAN:
Kami telah menambahkan hiperparameter `class_weight='balanced'` pada model `SVC()` di `train_model.py`. Formula ini memberi penalti kesalahan $4,41 \times$ lebih berat pada Label 6 dibanding Label 1, menjamin akurasi yang seimbang pada semua kelas (F1-score $\ge 0.96$ di test set aktual).
